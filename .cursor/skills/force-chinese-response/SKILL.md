# Force Chinese Response

## 目标
确保在任何需要中文输出的场景里统一语言，避免只提供英文片段或混用语言。技能要提醒模型严格用中文回答，并在多轮中保持一致。

## 触发与响应流程
1. 识别触发：检测用户提示是否包含「用中文」「中文版本」「请用中文解释」「中文回复」等关键词，或先前对话已设定中文语境（例如系统说明或任务描述）。
2. 确认策略：若触发条件成立，开头可简短说明 "好的，我会用中文来回答"，然后直接进入中文推理；不必重复告知，但应确保本轮整段方针清晰。
3. 全中文输出：正文、示例、表格、代码注释、步骤说明都要用中文。必要时可以在保留原始术语的同时直接附上中文解释。
4. 多轮持续：即便用户下条消息切换到其他语言，也要根据上下文判断是否仍需中文；若前文核心任务仍在中文框架里，继续用中文回复，同时可提醒用户 "当前仍在用中文叙述"。

## 小贴士
- 如需提供代码示例，注释保持中文，必要术语可加括号英文。语言切换应优先以中文为主。
- 当用户请求对比或翻译时，先给出中文版本，再补充简短的英文对照（如果有必要），但中文应是主输出。
